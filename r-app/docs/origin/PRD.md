# ccNexus 重构产品需求文档 (PRD)

## 文档说明

本文档是基于旧版 ccNexus 项目（`E:\myCode\localAway\ccNexus`）的重构计划。旧版项目作为参考实现，本 PRD 定义了重构后新系统的功能需求。

**旧版参考**
- 项目路径：`E:\myCode\localAway\ccNexus`
- 技术栈：Go + Wails v2 + Vue.js + SQLite（旧版技术栈，重构时待定）
- 主要功能：多端点轮换、API 格式转换、实时统计、WebDAV 同步

**重构目标**
- 保留核心功能，优化架构设计
- 提升代码可维护性和可扩展性
- 改善用户体验和性能
- 移除不需要的功能（Token Pool 管理等）

---

## Problem Statement

用户需要一个智能 API 端点轮换代理，用于 Claude Code 和 Codex CLI，解决以下问题：

1. **多端点管理困难**：需要手动切换不同的 API 端点，效率低下
2. **API 格式不兼容**：Claude、OpenAI、Gemini 等 API 格式不同，无法直接互换使用
3. **缺乏使用统计**：无法追踪各端点的使用情况和 Token 消耗
4. **配置同步困难**：多设备间配置和数据无法同步
5. **用户体验不足**：缺乏现代化的 GUI 界面和便捷的管理功能

## Solution

重构后的系统是一个智能 API 端点轮换代理，提供：

1. **自动端点轮换与故障转移**：智能管理多个 API 端点，自动故障转移
2. **API 格式转换**：支持 Claude ↔ OpenAI Chat 格式互转（后续可扩展其他格式）
3. **实时统计与监控**：四周期统计，事件驱动零延迟更新
4. **WebDAV 云同步**：多设备间同步配置和数据
5. **跨平台 GUI**：现代化桌面应用（具体技术栈待定）
6. **完善的管理功能**：端点筛选、克隆、测试、自动更新等

## Implementation Decisions

### 1. 架构设计

**运行模式**
- 桌面模式：跨平台 GUI 应用（Windows、macOS、Linux）
- 具体技术栈待定

**核心组件**
- 代理层：HTTP 代理核心，管理端点轮换和故障转移
- 转换器：API 格式转换（Claude ↔ OpenAI Chat），支持流式和非流式响应
- 存储层：SQLite 数据存储，支持并发访问
- WebDAV 客户端：云同步功能
- 前端：界面支持多语言和主题

**架构特点**
- 单一运行模式，简化部署和维护
- 桌面应用，用户友好
- 跨平台支持，覆盖主流操作系统

### 2. 数据库设计

**主要数据表**
- endpoints：端点配置
- endpoint_credentials：凭证管理
- credential_usage：凭证使用统计
- daily_stats：每日统计
- app_config：应用配置

**索引优化**
- 按日期、端点名、设备 ID 建立索引
- 支持高效的统计查询

### 3. 端点轮换策略

**轮换逻辑**
- 按顺序轮换：端点1 → 端点2 → 端点3 → ...
- 循环使用：最后一个端点失败后回到第一个
- 线程安全：使用锁保护并发访问

**重试策略**
- 最大重试次数 = 端点数量 × 2
- 同一端点最多连续失败 2 次后切换
- 网络瞬时错误重试同一端点（300ms 延迟）

### 4. API 格式转换

**支持的转换**
- Claude ↔ OpenAI Chat（核心功能）
- 后续可扩展：OpenAI Responses、Gemini 等格式

**转换特点**
- 双向支持
- 完整工具调用支持
- 流式响应支持
- 思考/推理内容支持

**扩展性设计**
- 转换器接口抽象，便于后续添加新格式
- 插件式架构，支持动态注册转换器
- 配置驱动，通过端点配置选择转换器

### 5. 统计系统

**四周期统计**
- 今日、昨日、本周、本月
- 事件驱动零延迟更新
- 趋势对比分析

**统计指标**
- 请求数、错误数
- 输入 Token 数、输出 Token 数
- 按端点统计

### 6. WebDAV 同步

**同步内容**
- 端点配置
- 应用设置
- 统计数据

**安全考虑**
- 过滤设备特定配置
- 元数据管理
- 连接测试
