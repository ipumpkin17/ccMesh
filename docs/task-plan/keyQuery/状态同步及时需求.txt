6:端点模型获取接口的问题。

在添加编辑端点的时候，拉取模型，存在一个问题，就是不同端点获取的模型不一样

当前是

┌─ 前端：端点表单「刷新」按钮
│
├─ 调用 fetch_endpoint_models(api_url, api_key, transformer)
│
└─ 后端 Rust
   │
   ├─ 1. 判断 transformer 格式 (Claude / OpenAI)
   │
   ├─ 2. 构建对应的 HTTP 请求
   │ ├─ Claude: x-api-key + anthropic-version
   │ └─ OpenAI: Authorization: Bearer
   │
   ├─ 3. 发送 GET /v1/models
   │
   ├─ 4. 解析 response.data[].id
   │ ├─ ✅ 成功: 返回模型 ID 列表
   │ └─ ❌ 失败: 返回 [默认模型]
   │
   └─ 前端显示模型列表

但是很多用户大概率不知道自己是什么端点，使用，需要聚合操作，不再区分ransformer 格式 (Claude / OpenAI)，如果用户选择的是Claude，优先使用Claude: x-api-key + anthropic-version构建请求模型，但是可能会存在获取不到模型列表的情况，这个时候需要多做一步，使用OpenAI: Authorization: Bearer构建再请求一次即可 。另外为了提高端点获取的成功率，需要采取

URL 候选构造策略，这个策略是在上述情况没法获取到模型资源才进行，获取到了就提前返回结构，否则才执行URL 候选构造策略，去获取模型资源

正确处理了不同格式的端点：

1. 用户提供的 url地址（精确覆写）
2. baseURL + /v1/models （剥离子路径）
3. 如果已以 /v{N} 结尾，则用 /models（这个就不处理了，因为任务5做了提示，还错就不应该了）
4. 剥离已知兼容子路径后再试（如剥离 /anthropic）
