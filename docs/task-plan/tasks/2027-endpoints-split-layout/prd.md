# 端点管理页上下分栏按视口比例布局

## Goal

端点管理页改为「固定头部 + 上下两个按视口高度比例分配的滚动区」：标题与筛选栏固定顶部；下方端点列表(上)与可用模型(下)按 60%/40% 分配剩余视口高度，各区内容超出时各自显示滚动条，整页不再作为单一长滚动。

## Requirements

- 标题「端点管理」+ FilterBar 固定在顶部，不随内容滚动。
- 端点列表区(上) : 可用模型区(下) = 60% : 40%，按剩余视口高度比例分配（非固定 px）。
- 任一区内容超出其高度时，该区内部出现纵向滚动条；另一区不受影响。
- 整页占满主内容区高度，不再产生整页滚动。

## Acceptance Criteria

- [ ] 端点很多时，端点区内部滚动，模型区位置/高度不变。
- [ ] 模型很多时，模型区内部滚动，端点区不变。
- [ ] 缩放窗口高度时上下区按 60/40 同步伸缩。
- [ ] 标题与筛选栏始终可见。

## Definition of Done

- `npx tsc --noEmit` 通过；现有前端测试不回归。
- 视觉/滚动行为无法无头验证，显式声明需本地确认。

## User Stories

- 作为用户，我希望端点和可用模型在一屏内上下分区各自滚动，以便对照查看而不必整页上下翻。

## Implementation Decisions

- 用 flex 比例 `flex-[3]`/`flex-[2]`（=3:2=60:40）分配头部之外的剩余空间，等效"按视口高度比例"，且自动扣除固定头部高度（优于硬编码 60vh/40vh 会溢出）。
- 滚动区用 `min-h-0 overflow-y-auto` 保证可收缩并内部滚动。
- 仅改 Endpoints 页根布局，不动 AppLayout（main 已是 `overflow-y-auto`，Tauri WebView2/Chromium 下 `h-full` 百分比高度可正常解析）。

## Out of Scope

- AppLayout / 其它页面布局。
- 端点稀少时上区留白属固定比例的预期表现，不做自适应压缩。

## Technical Notes

- 滚动条与卡片间距用 `pr-1` 留白。
