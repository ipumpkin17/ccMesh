# 稳定端点唯一 ID

## 问题

端点名称同时承担展示和身份用途。端点从 A 改名为 B 后，模型列表、运行状态和统计会把 B 当作新的身份；配置导入、快速队列和不同 CLI 的运行状态也可能因名称复用而串线。

## 决策

- SQLite 数值 `endpoints.id` 继续作为本机 CRUD 主键
- 新增 UUID `endpoints.uid` 作为跨模块稳定身份
- 名称只用于展示和当前快照，改名不改变 UUID
- 配置 v1 新增可选 `id` 字段；旧配置缺失 ID 时导入自动生成
- 新配置导入导出保留 UUID；非法/重复 UUID 明确报错
- 两台旧设备升级后出现同名不同 UUID 时保留本地身份，并显式计入导入结果
- 代理轮换、熔断、在途请求、当前端点和模型列表使用 UUID
- 新统计按 UUID 聚合、筛选和删除；旧统计 `endpoint_id` 留空，不做回填
- WebDAV 先升级临时备份结构，再按 UUID 合并；同名跨设备端点映射到本地 UUID

## 验收标准

- 端点改名后 UUID、轮换状态、熔断状态和当日统计身份保持不变
- `/v1/models` 的 `endpoint_id` 返回 UUID，`owned_by` 返回展示名称
- 用户提供的无 ID v1 配置可直接导入并自动生成 UUID
- 新导出的配置包含 UUID，再导入后保持一致
- 同名不同 UUID 保留本地 UUID，远端新统计同步映射到该身份
- Dashboard 当前端点和健康态按 UUID 高亮
