# 应用更新提示与导航栏图标置换

## Goal

把导航栏左上角的占位图标换成真实品牌图标；并把"已检测到更新却无明细提示、无便捷更新入口"的现状补全为：常驻版本号 + 按需出现的更新图标（点击即更新）+ 点击版本号弹出的版本信息浮层（更新日志/手动检查/发布页跳转/Star）。

## Requirements

1. 导航栏 Logo 使用真实图片资源（来自应用图标集），替换当前的占位矢量图标；侧栏与顶栏两种导航模式一致生效，折叠态只显示图标。
2. 导航栏常驻显示当前应用版本号（本地读取，不依赖联网检查）。
3. 仅当检测到可用更新时，在版本号旁显示「更新图标」；点击更新图标直接开始下载并安装。
4. 单击版本号弹出「版本信息」浮层（锚定在版本号附近的小卡片），包含：
   - 标题区 + 右上角「手动检查」按钮（重新联网检查更新）；
   - 大号当前版本号 + 状态（已是最新 / 发现新版本 vX）；
   - 更新日志正文（有则展示，来自检查结果）；
   - 「查看发布」入口（带 GitHub 标识）跳转到 GitHub 发布页；
   - 一个填充高亮的 Star 图标按钮，点击同样跳转到 GitHub 发布页。
5. 所有 GitHub 跳转用系统默认浏览器打开外部链接。

## Acceptance Criteria

- [ ] 导航栏图标显示为真实品牌图片（非 ZapIcon），侧栏展开/折叠、顶栏模式均正确。
- [ ] 导航栏显示形如 `v0.1.2` 的当前版本号，与应用打包版本一致。
- [ ] 无可用更新时不显示更新图标；有可用更新时显示，点击后触发下载安装并给出反馈。
- [ ] 单击版本号弹出 Popover 浮层；点浮层外自动关闭。
- [ ] 浮层「手动检查」可重新检查并刷新浮层内状态/版本/日志；最新时给出"已是最新"反馈。
- [ ] 浮层「查看发布」与 Star 图标点击，均用系统浏览器打开 GitHub 发布页。
- [ ] `pnpm check`（tsc + cargo check）与 `pnpm test` 通过。

## Definition of Done

- 上述验收标准全部满足；新增的外部链接打开能力具备明确的 capability 范围限制（只放行本项目 GitHub 地址）。
- 版本信息浮层的更新检查/下载逻辑复用既有更新链路（不新增重复后端命令）。
- 无法在无头环境验证的部分（真实出网检查更新、系统浏览器跳转、下载安装与重启）以核对清单形式显式声明。

## User Stories

- 作为用户，我希望导航栏显示真实品牌图标与当前版本号，以便一眼确认所用应用与版本。
- 作为用户，我希望仅在有更新时才看到更新图标并能一键更新，以便不被无谓提示打扰、又能及时升级。
- 作为用户，我希望单击版本号就能看到更新日志、手动检查并跳转发布页，以便了解版本详情与获取发布。

## Implementation Decisions

- **弹窗形态**：Popover 锚定浮层（非居中 Dialog），点外部关闭。
- **更新图标点击**：直接调用既有「下载并安装」流程，不二次弹窗确认；以 toast/进度给出反馈。
- **Star 图标行为**：可点击，跳转 GitHub **发布页**（与"查看发布"同目标，不跳仓库主页）。
- **名称文案**：保持 `ccMesh`（与 productName、窗口标题一致；参考截图中的 "Code AI" 仅取其布局）。
- **版本号来源**：前端用应用运行时 API 本地读取版本（不联网、启动即显示）；经核实当前权限集已覆盖，无需新增权限。
- **外部链接**：引入官方 opener 能力打开系统浏览器；capability 仅放行本项目 GitHub 地址范围。
- **复用**：版本检查/下载/进度/日志逻辑沿用既有更新服务与 store，不新增后端命令。

## Testing Decisions

- 以组件测试覆盖可纯前端验证的行为：更新图标按更新可用状态显隐、浮层状态文案（最新/有新版本）随检查结果切换。tauri 侧 API（版本读取、检查更新、打开链接）以 mock 注入。
- 真实出网检查、下载安装、系统浏览器跳转属环境依赖，归入手动核对清单。

## Out of Scope

- 更新设置项（自动检查开关/间隔）的改动——沿用 Settings 页既有实现。
- 安装完成后的自动重启/relaunch 行为——保持与现有「下载并安装」一致，不在本次引入。
- 应用图标集本身的重制（仅复用现有 `Square71x71Logo.png`）。
- 顶栏导航模式下版本浮层的布局精修——以侧栏为主，顶栏保证可用即可。

## Technical Notes

- 当前更新链路已具备：检查更新、下载并安装（带进度事件）、跳过版本、读写更新设置；本次只在前端补 UI 与交互入口。
- 打开外部链接此前无任何手段（未装 opener/shell，CSP 未放行），属本次必须新增的基础能力。
- GitHub 发布页地址由 updater endpoint 推导（同一仓库的 /releases）。
