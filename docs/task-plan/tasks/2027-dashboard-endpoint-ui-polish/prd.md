# 仪表盘 / 端点 / 设置 多项 UI 与熔断优化

## Goal

按 `需求.txt` 完成 6 项体验优化：仪表盘卡片与监控展示更易读、请求明细补充耗时/首字、端点编辑可查看密钥并修复 JSON 编辑器错位、端点熔断按"该端点是否拥有该模型"隔离、设置页排版美化。

## Requirements

1. 仪表盘「Token（今日）」卡片：换算单位小字改为数值下方垂直布局。
2. 实时请求监控悬停 Token 卡：超过 1000 的数值用 `1k/2k/110k` 表示；耗时按秒展示（`6.46s`）。
3. 请求明细表新增「用时」「首字」两列；首字 = 上游首字节到达耗时（流式取首个内容分片，缓冲取响应头到达）。
4. 端点编辑表单：API Key 支持显隐切换查看；修复 JSON 编辑器布局错位。
5. 端点自动切换/熔断：仅在「声明了该请求模型」的端点间轮换与熔断；无任一端点声明该模型时回退全量（向后兼容）。
6. 设置页「系统 / 高级」与首段排版美化，向"代理"段的标题+副述+narrow input 视觉风格对齐。

## Acceptance Criteria

- [ ] Token 卡片小字位于数值下方。
- [ ] 悬停 Token 卡：`102291 → 102k`、`1025 → 1k`、`94 → 94`；耗时 `6458ms → 6.46s`。
- [ ] 明细表出现「用时」「首字」列；首字对旧行/无数据显示 `—`。
- [ ] 编辑端点时可点击眼睛图标查看 API Key 明文。
- [ ] 请求 `claude-opus-4-8`（仅 max/max2 声明）不会路由到只含 `mimo` 的 cc 端点，cc 熔断不受影响。
- [ ] 设置页系统/高级段与代理段视觉风格一致。

## Definition of Done

- `cargo test`（后端）与 `npx vitest run` / `npx tsc --noEmit`（前端）通过。
- 新增逻辑（模型过滤、k/秒格式化）有单元测试。
- 无法无头验证的视觉项（JSON 编辑器对齐、设置页美观）显式声明需本地确认。

## User Stories

- 作为用户，我希望大额 Token 一眼可读（k 单位）且耗时以秒计，以便快速判断请求成本与延迟。
- 作为用户，我希望看到每次请求的用时与首字延迟，以便评估端点响应速度。
- 作为用户，我希望编辑端点时能核对已保存的密钥，以便排查鉴权问题。
- 作为运维，我希望某模型在其专属端点熔断时不连累其它端点的其它模型，以便故障隔离精准。

## Implementation Decisions

- 任务 5 取"按模型过滤候选端点"方案（最小且匹配示例的跨端点隔离），不改熔断器键为 (端点,模型)；模型未被任何端点声明时回退全量以保向后兼容。
- 首字延迟新增 `first_byte_ms` 列（v6 迁移），贯穿 RequestRecord→RequestLog→repo→前端。
- Token k 格式化取整数 k（`Math.round(n/1000)`），<1000 显示原值；耗时统一 `(ms/1000).toFixed(2)s`。

## Testing Decisions

- 模型过滤：在 resolver 增 `filter_by_model` + 单测（声明/未声明/大小写）。
- 格式化：在 `lib/format.ts` 增 `formatTokenK`/`formatDuration` + vitest。
- 迁移：v6 列存在性单测。

## Out of Scope

- 熔断器按 (端点,模型) 维度细分（本期仅跨端点隔离）。
- 历史旧行回填 first_byte。

## Technical Notes

- JSON 编辑器错位与设置页美观属视觉项，无头环境无法验证，改完需本地核对。
